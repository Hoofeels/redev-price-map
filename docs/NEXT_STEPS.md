## A. 재개발 빌라·단독 라이브 실거래가 — ✅ 완료 (rh/sh 승인됨 2026-06-21)

**결과:** 사용자 `getRTMSDataSvcRHTrade`/`getRTMSDataSvcSHTrade` 활용신청 완료(같은 일반인증키).
라이브 검증: 의정부 장암5(재개발) matched **401 = 아파트 267 + 빌라·단독 134**, `skippedKinds=[]`.
코드 변경 불필요(기존 `kinds=["rh","sh","apt"]` + 동일 키). 아래는 기록용.

**(완료된) 신청 절차:**

**해야 할 일 (data.go.kr, 무료, 보통 1~2시간 내 자동승인):**
1. 기존 `MOLIT_API_KEY`를 발급한 **같은 data.go.kr 계정**으로 로그인.
2. 아래 두 API를 **활용신청**(검색창에 서비스명 입력):
   - `국토교통부_연립다세대 매매 실거래가 자료` → 오퍼레이션 `getRTMSDataSvcRHTrade`
   - `국토교통부_단독/다가구 매매 실거래가 자료` → 오퍼레이션 `getRTMSDataSvcSHTrade`
3. 승인되면 **추가 코드/설정 변경 불필요** — 같은 키라서 자동으로 빌라·단독 거래가 들어옵니다.

**검증:** 재개발 zone으로 호출해 `skippedKinds`에 `rh`/`sh`가 사라지면 성공
```
GET /api/transactions?zoneId=<재개발 zoneId>&months=12   → skippedKinds: []  (이전엔 ["rh","sh"])
```

## D. 지오코딩 보조(Kakao 폴백) — ✅ 완료 (2026-06-21)

> Kakao REST 키 입력 + 카카오맵(Local) 서비스 활성화 후 `npm run geocode` 재실행 → **280→282 geocoded**.
> Kakao가 해소: 강서 내발산동 716-20 (37.5494,126.8337), 서초 방배동 593-98 (37.4747,126.9933).
> **남은 3건(시군구 중심 근사 유지, 무해):**
> - 노원 공릉동 154-3 — VWorld+Kakao 모두 NOT_FOUND → 정보몽땅 **대표지번 부정확/부재** 가능성(데이터 품질).
> - 노원 상계동 자력2구역 17블럭 1롯트 / 자력6구역 8블럭 9롯트 — 구역 내부 블럭·롯트 표기로 **실제 지번 아님 → 좌표화 불가**.
>
> 코드: `lib/geocode.ts` `geocodeKakao`(주소→키워드 2단) + `geocodeBest(vworldKey,address,kakaoKey?)`,
> `scripts/geocode.ts`가 `KAKAO_REST_KEY` 감지 시 VWorld 실패(null)만 Kakao 재시도(캐시 성공분 보존).
> ⚠️ Kakao 앱은 **제품 설정 > 카카오맵 활성화 ON** 필요(REST 키만으론 403 `OPEN_MAP_AND_LOCAL disabled`).

**대상(VWorld NOT_FOUND, 현재 시군구 중심 근사 폴백):**
- 노원 공릉동 154-3
- 강서 내발산동 716-20
- 서초 방배동 593-98

**옵션:**
- **(권장) Kakao 주소검색 API** — `developers.kakao.com`에서 REST 키 무료 발급 → `.env.local`에 `KAKAO_REST_KEY=` 추가.
  지번/도로명 검색이 강건해 위 3건 해소 가능성 높음. 키 주시면 `lib/geocode.ts`에 Kakao 폴백(VWorld 실패 시)을 제가 추가합니다.
- **국토부 지오코더**(`data.go.kr` 15101106) 활용신청 — 공공데이터 계정으로 키 발급 후 동일하게 폴백 연결 가능.
- **그대로 두기** — 3건만 시군구 중심 근사. 지도 표시·필터·매칭에 영향 없음(무해).

> ⚠️ 주의: Kakao/VWorld **지오코더 결과 좌표를 저장/캐싱하는 것은 각 ToS상 제약**이 있습니다(M0.5 조사).
> 현재 앱은 빌드 시 1회 지오코딩해 `data/geocode-cache.json`에 저장하는 구조이므로,
> 공개 배포·상업화 시에는 국토부 지오코더(공공데이터, 저장 허용) 또는 폴리곤 centroid 사용을 권장합니다.

## 기타 한계 (참고)

- **입주권 프리미엄/권리가:** 공개 데이터에 **없음**(국토부 분양·입주권 API는 거래금액만, 프리미엄/권리가 미공개 — M0.5 조사 확정). UI/모델 구분은 구현돼 있으나 라이브 소싱 불가. AC11의 확정된 데이터 한계.
- **고양 등 일부 시 동→구 매핑:** 주소에 자치구가 없는 구역(예: 고양 행신동)은 시-단위 LAWD로 폴백 → MOLIT 0건. 동→구 매핑 테이블을 추가하면 해소 가능(소규모 과제).
- **관리처분/철거 단계 구역:** 이주·철거가 진행돼 최근 실거래가 거의 없을 수 있음(데이터 현실, 버그 아님).

## 네이버 호가(참고) — `scrape:naver` (로컬 전용, ToS 주의)

PoC 검증 완료(Playwright 실브라우저로 429 우회, 단지 호가 캡처). 상세패널에 "호가(참고)" 섹션으로 표시.

**사용:**
```
npm run scrape:naver -- --limit=200     # 재건축+대표단지 N개 수집
npm run scrape:naver -- --zone=seoul-11170-14   # 특정 구역만
```
→ `data/naver-asking.json`에 zone별 호가 저장. 상세패널이 자동 표시(재건축 단지 위주, 재개발은 단지 부재로 제한).

**⚠️ ToS / 커밋 주의:**
- 네이버부동산은 공식 API 없음 → 실브라우저 best-effort. **네이버 변경 시 깨질 수 있음.**
- `data/naver-asking.json`은 **빈 `[]`로 커밋**되어 있고, 수집 결과는 **로컬에만** 둔다(네이버 호가를 git 히스토리/공개 원격에 넣지 말 것).
- 커밋 사고 방지: `git update-index --skip-worktree data/naver-asking.json` (로컬 변경 추적 제외). 되돌리려면 `--no-skip-worktree`.
- 공개 배포 시 호가 재호스팅은 ToS 위반 → 로컬 분석/조회용으로만.

**해소(자동화):** 매물명 정규화·지역 가드로 동명이단지 오매칭 방지(예: "산호"→칠곡산호한양 걸러 용산 산호=759 채택). 미해소: 단지명이 네이버와 다른 구역(예: "강변강서")은 매칭 실패 → 수동 complexNo 매핑으로 보완 가능.
